1. The problem we're solving

Imagine the Titanic dataset. For each passenger we know some facts — age, fare paid, whether they were in first class — and we want to predict whether they survived.

agefareis_first_class→survived
227.250→0
3871.281→1
267.920→1
3553.101→1

A decision tree is a flowchart of yes/no questions that ends in an answer. For example: 

is_first_class ≤ 0.5?
├─ yes → age ≤ 24?
│        ├─ yes → predict 0 (did not survive)
│        └─ no  → predict 1 (survived)
└─ no  → predict 1 (survived)

The tree's job, given a table of training examples, is to figure out these questions and their order by itself. That learning process is what all this code does.

Why a tree? Because it's interpretable and reasonably powerful. You can print the tree and read off exactly why a prediction was made. Random Forests and XGBoost — two of the most common winners on tabular data — are built out of many such trees.

2. Vocabulary: data and labels

Hard-coded in your brain before any code makes sense:

  • sample (also row, observation, example): one individual thing you have data about. In the table above, one passenger.
  • feature (also column, attribute, variable): one piece of information per sample. age is a feature. fare is a feature. A sample has many features.
  • label (also target, y, class): the thing we want to predict. For Titanic, survived. There's exactly one label per sample.
  • X: the whole table of features. Every row is one sample, every column is one feature. In code: std::vector<std::vector<double>> — a vector of rows, each row a vector of numbers.
  • y: the list of labels, one per sample. std::vector<double>. It's flat (1D) because there's just one label per row.
  • training: the one-time process of looking at lots of labeled data and learning a tree from it.
  • prediction (also inference): using the already-trained tree to guess the label for a new sample whose label we don't know.
Anchor X.size() = number of samples (rows).
X[0].size() = number of features (columns).
X[i][j] = feature j of sample i.
y[i] = label of sample i.
Always, always X.size() == y.size().

3. Vocabulary: the tree itself

  • node: one box in the flowchart. Can be internal or a leaf.
  • internal node: a box that asks a question (is feature j ≤ threshold?) and has two children.
  • leaf (or leaf node): a box at the bottom that gives an answer — no more questions.
  • root: the topmost node. Every prediction starts here.
  • child: the two nodes that an internal node points to. Traditionally called left_child (the "yes" branch, feature ≤ threshold) and right_child (the "no" branch, feature > threshold).
  • depth: how many questions deep you are. Root is depth 0, its children are depth 1, and so on. Capping depth prevents the tree from becoming giant and memorizing the training data.
  • threshold: the numeric cutoff in a question. For age ≤ 24.5?, the threshold is 24.5.
  • feature_index: which column the question is about. If age is column 0 and the question is age ≤ 24.5?, then feature_index = 0.
  • split: the act of partitioning a group of samples in two, based on a (feature_index, threshold) pair.
  • impurity: a number that says how mixed-up the labels in a group are. 0 means "all the same class" (pure). Higher means more mixed. Your code uses the Gini impurity.

4. Vocabulary: the C++ bits you'll see

Not a C++ course — just the concepts this file actually uses, in the order they appear. Skim and refer back.

#pragma once

A one-liner that says "only include this header once per compilation unit." Prevents the code from being pasted twice, which would cause duplicate definitions.

#include <…>

Copy-paste of another header's contents. <vector> brings in std::vector. <map> brings in std::map. And so on.

std::vector<T>

A dynamically-resizing array of T. std::vector<double> is an array of doubles. std::vector<std::vector<double>> is an array of arrays of doubles — a 2D table.

std::map<K, V>

An ordered dictionary: keys of type K mapping to values of type V. std::map<double, int> means "for each unique double key, store an int." Perfect for counting: counts[label]++ adds 1 to the count of that label, creating the entry first if needed.

std::unique_ptr<T>

A pointer that owns what it points to. When the unique_ptr goes out of scope, it automatically deletes the thing. You never call delete. "Unique" means no two unique_ptrs can point at the same object — ownership is single. For a tree, this means: when the root dies, its children die, their children die, the whole subtree cleans itself up.

std::make_unique<T>()

The safe way to create a unique_ptr<T>. std::make_unique<Node>() allocates a new Node and wraps a unique_ptr around it.

std::pair<A, B>

A two-element tuple. Used here as the return type when a function needs to hand back two things at once (the split functions return both the resulting rows and the resulting labels).

References: T& and const T&

T& means "a reference to a T" — essentially an alias for an existing object, no copy made. const T& means "a read-only alias." Passing large objects like vectors by const T& avoids copies while still preventing accidental modification.

static member function

A function that belongs to the class but doesn't need an instance to be called. Called as DecisionTree::gini(labels) rather than tree.gini(labels). Used for utilities that don't read or write member variables (max_depth_ etc).

auto

"Let the compiler figure out the type." auto size = labels.size(); — the compiler knows labels.size() returns size_t, so size becomes size_t.

size_t

An unsigned integer big enough to count anything you could fit in memory. std::vector uses it for sizes and indices. The gotcha: because it's unsigned, 0 - 1 underflows to a huge number instead of becoming -1.

Structured bindings: auto [a, b] = pair;

A way to unpack a pair (or tuple, or struct) into named variables in one line. Instead of auto result = splitLeft(...); auto X_left = result.first; auto y_left = result.second; you get auto [X_left, y_left] = splitLeft(...);. Cleaner.

Lambdas: [](const auto& a, const auto& b) { return a.second < b.second; }

An anonymous function, defined inline. The [] is the capture list (empty here, meaning the lambda doesn't need any surrounding variables). The part in parentheses is the parameter list. The body is after. Used to tell algorithms like max_element how to compare things.

static_cast<T>(x)

An explicit type conversion. static_cast<double>(labels.size()) turns a size_t into a double so division doesn't do integer truncation.

[[nodiscard]]

An attribute that tells the compiler: "if someone calls this function and throws the return value away, warn them." Useful on functions whose whole purpose is the return value, like predict.

assert(condition && "message")

A runtime check: if condition is false, the program aborts with a readable error. The && "message" trick is a C idiom — a non-empty string is truthy, so the message doesn't change the logic, but it does show up in the error output.

5.3. The Node struct (lines 12–19)

struct Node {
    int feature_index = -1;
    double threshold = 0.0;
    double prediction = 0.0;
    bool is_leaf = false;
    std::unique_ptr<Node> left_child;
    std::unique_ptr<Node> right_child;
};

One Node is one box in the flowchart. It has two personalities:

  • Internal node (is_leaf == false): uses feature_index and threshold to decide which child to visit. prediction is unused.
  • Leaf node (is_leaf == true): just outputs prediction. feature_index, threshold, and the children are unused.

The = -1, = 0.0 etc. are default member initializers. A freshly-made Node starts out with those values; you set them later. The unique_ptrs default-construct to null (no child pointed at), which is exactly what a leaf should have.

5.4. The SplitResult struct (lines 20–25)

struct SplitResult {
    int feature_index = -1;
    double threshold = 0.0;
    double score = 1.0;
    bool valid = false;
};

While searching for the best question to ask, we need to remember the best candidate seen so far: which feature, which threshold, and how good it was. This is that memo.

  • score starts at 1.0 because Gini is always between 0 and 1, and lower is better — any real split will beat 1.0.
  • valid starts at false: if no real split is ever found (can happen on degenerate data), the caller can detect it.

5.5. The instance variables (lines 26–28)

int max_depth_;
int min_samples_;
std::unique_ptr<Node> root_;
  • max_depth_: hard cap on how deep the tree can grow. If allowed to grow forever, the tree will memorize every training sample (overfitting) and perform poorly on new data.
  • min_samples_: if a group has fewer than this many samples, stop splitting — the node becomes a leaf. Prevents silly splits on tiny groups.
  • root_: the trained tree. Starts null. After fit(), it holds the actual tree. The trailing underscore is a convention signaling "member variable," which avoids name clashes with constructor parameters.

5.6. traverse (lines 30–36)

static double traverse(const Node* node, const std::vector<double>& sample) {
    if (node->is_leaf) return node->prediction;
    if (sample[node->feature_index] <= node->threshold) {
        return traverse(node->left_child.get(), sample);
    }
    return traverse(node->right_child.get(), sample);
}

Given one sample and a tree (passed as a raw pointer to its current node), walk down the tree and return the leaf's prediction. This is the prediction-time counterpart to buildTree.

Step by step:

  1. If this node is a leaf, return its prediction — we're done.
  2. Otherwise look at feature feature_index of the sample.
  3. If that value is ≤ the threshold, recurse into the left child.
  4. Otherwise recurse into the right child.

.get() on a unique_ptr returns the raw pointer inside (without giving up ownership). We use a raw pointer here because traverse only needs to look at nodes, not own them.

5.7. gini (lines 38–49)

static double gini(const std::vector<double>& labels) {
    if (labels.empty()) return 0.0;
    std::map<double,int> unique;
    for (auto label : labels) unique[label]++;
    const auto size = static_cast<double>(labels.size());
    double sum = 0.0;
    for (const auto val: unique | std::views::values) {
        const auto p = val / size;
        sum += p * p;
    }
    return 1.0 - sum;
}

Gini impurity measures how mixed a group of labels is. Formula: 1 - Σ pᵢ², where pᵢ is the fraction of samples in class i.

  • All one class: p = 1, 1 - 1² = 0. Pure.
  • Binary 50/50: two classes at p = 0.5, 1 - (0.25 + 0.25) = 0.5. Max mess.
  • Binary 90/10: 1 - (0.81 + 0.01) = 0.18. Mostly pure.

Line-by-line:

  • if (labels.empty()) — guard against 0/0 below.
  • std::map<double,int> unique — counter: label → count.
  • for (auto label : labels) unique[label]++; — tally. Accessing unique[label] for the first time creates the entry initialized to 0, then ++ makes it 1.
  • static_cast<double>(labels.size()) — convert to double so val / size below does floating-point division, not integer division.
  • unique | std::views::values — a range view. It gives you just the values of the map (the counts), not the keys. You don't need the label values themselves here, only their counts.
  • p = val / size; sum += p * p; — accumulate Σ pᵢ².
  • return 1.0 - sum; — the Gini.

5.8. weightedGini (lines 50–56)

static double weightedGini(const std::vector<double>& left_labels,
                           const std::vector<double>& right_labels) {
    const auto n_total = static_cast<double>(left_labels.size() + right_labels.size());
    if (n_total == 0) return 0.0;
    const auto w_left  = static_cast<double>(left_labels.size())  / n_total;
    const auto w_right = static_cast<double>(right_labels.size()) / n_total;
    return w_left * gini(left_labels) + w_right * gini(right_labels);
}

After a split, you have two groups. This function gives the combined impurity of both — but weighted by group size, so a messy group of 500 matters more than a pure group of 2.

Why weighted? If you averaged the two Ginis equally, you'd be tricked: a split that puts 1 sample on one side (trivially pure, gini 0) and 999 mixed samples on the other (gini 0.5) would look as good as splitting 500/500 with gini 0.25 on each. Weighting penalizes lopsided splits properly.

5.9. splitLeft and splitRight (lines 57–88)

static std::pair<std::vector<std::vector<double>>, std::vector<double>>
splitLeft(const std::vector<std::vector<double>>& X,
          const std::vector<double>& y,
          const int feature_index,
          const double threshold) {
    std::vector<std::vector<double>> X_out;
    std::vector<double> y_out;
    for (int i = 0; i < X.size(); i++) {
        if (X[i][feature_index] <= threshold) {
            X_out.push_back(X[i]);
            y_out.push_back(y[i]);
        }
    }
    return std::make_pair(X_out, y_out);
}

Given a feature and threshold, walk all the rows. For each row i: if X[i][feature_index] ≤ threshold, copy that row (and its matching label) into the output. splitRight is the same but uses >.

Critical invariant Every time you push X[i] you must also push y[i]. Lose that pairing and labels no longer match features — total garbage. That's why the function returns a pair of both rather than two separate functions.

The return type looks intimidating but it's just "a pair of (rows, labels)." std::make_pair builds that pair.

5.10. bestSplit (lines 89–113)

The heart of training. Try every reasonable question, score each one, return the best.

static SplitResult bestSplit(const std::vector<std::vector<double>>& X,
                             const std::vector<double>& y) {
    SplitResult best;
    if (X.empty() || X[0].empty()) return best;
    const int n_features = static_cast<int>(X[0].size());
    for (int j = 0; j < n_features; j++) {
        std::vector<double> vals;
        vals.reserve(X.size());
        for (const auto& row : X) vals.push_back(row[j]);
        std::ranges::sort(vals);
        vals.erase(std::ranges::unique(vals).begin(), vals.end());
        for (size_t k = 0; k + 1 < vals.size(); k++) {
            const double threshold = (vals[k] + vals[k + 1]) / 2.0;
            auto [X_left,  y_left]  = splitLeft(X, y, j, threshold);
            auto [X_right, y_right] = splitRight(X, y, j, threshold);
            if (y_left.empty() || y_right.empty()) continue;
            if (const double score = weightedGini(y_left, y_right); score < best.score) {
                best.feature_index = j;
                best.threshold = threshold;
                best.score = score;
                best.valid = true;
            }
        }
    }
    return best;
}

Outer loop: for each feature j

X[0].size() gives the number of columns. X.size() would give rows — wrong. Every row has the same width, so we pick the first.

Building candidate thresholds

  1. vals collects every value in column j, one per row. reserve(X.size()) pre-allocates memory so the push_backs don't keep reallocating.
  2. std::ranges::sort(vals) — sorts ascending.
  3. std::ranges::unique(vals) — shuffles duplicates to the end and returns an iterator to the new "logical end." We then erase from that point to the actual end. Classic "erase-remove" idiom. We dedup because splitting between two equal values makes no sense.
  4. threshold = (vals[k] + vals[k + 1]) / 2.0 — the midpoint between two consecutive unique values. Using the midpoint (instead of, say, just vals[k]) puts the threshold between the samples cleanly.
Loop condition gotcha The loop says k + 1 < vals.size(), not k < vals.size() - 1. If vals is empty, vals.size() - 1 underflows to a gigantic number and the loop never ends. Writing k + 1 < ... sidesteps that.

Scoring each candidate

  • Call splitLeft and splitRight, unpack with structured bindings.
  • If either side is empty, this threshold is degenerate — skip it.
  • Compute weightedGini. If it beats the current best.score, save everything including valid = true.

The if (const double score = weightedGini(...); score < best.score) syntax is the init-statement form of if: it defines score inside the if's scope so you can't accidentally use it outside.

Complexity

O(features × samples × samples): up to samples thresholds per feature, each requiring an O(samples) partition. Fine for Titanic. Real libraries sort once and sweep for much better asymptotics; you can add that later.

5.11. leafPrediction (lines 114–120)

static double leafPrediction(const std::vector<double>& labels) {
    std::map<double, int> counts;
    for (const auto label : labels) counts[label]++;
    const auto winner = std::ranges::max_element(counts,
        [](const auto& a, const auto& b) { return a.second < b.second; });
    return winner->first;
}

When a node becomes a leaf, we need one label to output. The simplest choice: majority vote — whichever class shows up the most among the samples that landed in this leaf.

  • Same map trick as gini — tally labels.
  • std::ranges::max_element(counts, cmp) returns an iterator to the largest entry. The default comparison on pair<K, V> would sort by key (the label value) — wrong. So we pass a lambda that compares by .second (the count).
  • The lambda uses < (less-than) because max_element wants a "strictly less" comparator: the element for which nothing is strictly greater wins.
  • winner->first gives the label (the map's key).
Tie behavior If two classes are tied, max_element returns the first one encountered. Since std::map iterates in sorted key order, that's the class with the smaller numeric label. It's deterministic — which is what you want.

5.12. buildTree (lines 121–150)

The recursive function that grows the tree. Given a slice of data and a depth, it either returns a leaf or returns an internal node whose children are built by recursive calls. 

static std::unique_ptr<Node> buildTree(
    const std::vector<std::vector<double>>& X,
    const std::vector<double>& y,
    const int depth,
    const int max_depth,
    const int min_samples) {

    if (depth >= max_depth
        || y.size() < static_cast<size_t>(min_samples)
        || gini(y) == 0.0) {
        auto leaf = std::make_unique<Node>();
        leaf->is_leaf = true;
        leaf->prediction = leafPrediction(y);
        return leaf;
    }
    const SplitResult best = bestSplit(X, y);
    if (!best.valid) {
        auto leaf = std::make_unique<Node>();
        leaf->is_leaf = true;
        leaf->prediction = leafPrediction(y);
        return leaf;
    }
    auto [X_left,  y_left]  = splitLeft (X, y, best.feature_index, best.threshold);
    auto [X_right, y_right] = splitRight(X, y, best.feature_index, best.threshold);
    auto node = std::make_unique<Node>();
    node->feature_index = best.feature_index;
    node->threshold     = best.threshold;
    node->left_child  = buildTree(X_left,  y_left,  depth + 1, max_depth, min_samples);
    node->right_child = buildTree(X_right, y_right, depth + 1, max_depth, min_samples);
    return node;
}

Stopping rules (the three base cases)

  1. depth >= max_depth — tree is tall enough; don't grow further.
  2. y.size() < min_samples — too few rows to justify splitting. Note the static_cast<size_t>(min_samples): size() is size_t; without the cast, comparing to an int triggers a signed/unsigned warning.
  3. gini(y) == 0.0 — node is already pure. Splitting would do nothing useful.

If any is true: make a leaf. A leaf is a Node with is_leaf = true and prediction set to the majority class.

Fourth base case: no valid split

Even if we didn't hit the three above, it's possible that every feature in this slice has only one unique value, so no threshold exists. bestSplit signals this by returning valid = false, and we make a leaf too.

Why order matters The three cheap stopping checks happen before we call bestSplit. That's important because bestSplit does X[0].size() — if X is empty, that's undefined behavior. The y.size() < min_samples check (with min_samples ≥ 1) stops us before we get there.

Recursive case

  1. Call splitLeft/splitRight with the best split to partition the data.
  2. Build a new internal Node. Set feature_index and threshold so traverse can route samples later. Leave is_leaf = false (default).
  3. Grow the left subtree recursively and attach it to node->left_child.
  4. Grow the right subtree and attach it to node->right_child.
  5. Return the finished node up to the caller.

Because the two children call buildTree with smaller data and one-deeper depth, the recursion always terminates — at worst, depth reaches max_depth and everything becomes a leaf.

5.13. Constructor (lines 151–152)

explicit DecisionTree(const int max_depth = 5, const int min_samples = 2)
    : max_depth_(max_depth), min_samples_(min_samples) {}
  • explicit: prevents implicit conversion. Without it, a stray DecisionTree t = 7; would quietly create a tree with max_depth = 7. With explicit, you must write DecisionTree t(7);. Safer.
  • Default arguments: = 5 and = 2 mean DecisionTree t; works with no arguments and you get reasonable defaults.
  • Member initializer list: the : max_depth_(max_depth), ... part sets the members directly instead of assigning inside {}. For int it's cosmetic. For class-type members it's the idiomatic way and can avoid an unnecessary default-construct-then-overwrite.
  • root_ isn't in the initializer list — unique_ptr defaults to null, which is exactly what "untrained" should look like.
  • Empty {} body: nothing else to do.

5.14. fit (lines 153–155)

void fit(const std::vector<std::vector<double>>& X,
         const std::vector<double>& y) {
    root_ = buildTree(X, y, 0, max_depth_, min_samples_);
}

The public "train me" function. Delegates all work to buildTree, passing depth 0 (we're starting at the root) and the instance's max_depth_ and min_samples_. The returned unique_ptr<Node> is moved into root_, which then owns the entire tree.

5.15. predict (lines 156–159)

[[nodiscard]] double predict(const std::vector<double>& sample) const {
    assert(root_ != nullptr && "DecisionTree is not fitted yet.");
    return traverse(root_.get(), sample);
}
  • [[nodiscard]]: warn if someone throws the return away.
  • const after the parameter list: this method doesn't modify any member variables. Lets you call predict on a const DecisionTree.
  • assert(root_ != nullptr && "…"): crash with a clear message if someone calls predict before fit. In release builds, assert is compiled out, so there's no runtime cost in production.
  • root_.get(): get the raw Node* inside the unique_ptr, without transferring ownership. traverse only needs to look.

6. Life of a prediction

Let's trace the whole system on a tiny dataset.

Input

// two features: [age, is_first_class]
// label: survived (0 or 1)
std::vector<std::vector<double>> X = {
    {22, 0}, {38, 1}, {26, 0}, {35, 1}, {8, 0}, {54, 1}
};
std::vector<double> y = { 0, 1, 1, 1, 0, 0 };

DecisionTree tree(3, 2);  // max_depth = 3, min_samples = 2
tree.fit(X, y);

Step 1: fit calls buildTree(X, y, 0, 3, 2)

  • depth (0) < max_depth (3) ✓
  • y.size() (6) ≥ min_samples (2) ✓
  • gini(y) on {0,1,1,1,0,0}: 3 zeros, 3 ones → 1 - (0.25+0.25) = 0.5. Not pure ✓
  • Call bestSplit(X, y).

Step 2: bestSplit tries every question

Feature 0 (age): unique sorted values = [8, 22, 26, 35, 38, 54]. Midpoints: [15, 24, 30.5, 36.5, 46].

Feature 1 (is_first_class): unique sorted values = [0, 1]. Midpoint: [0.5].

For each midpoint, split and score:

featurethresholdleft labelsright labelsweighted gini
age15{0}{0,1,1,1,0}1/6·0 + 5/6·(1-(0.4²+0.6²)) = 0.40
age24{0,0}{1,1,1,0}2/6·0 + 4/6·(1-(0.25²+0.75²))= 0.25
age30.5{0,1,0}{1,1,0}3/6·(0.44) + 3/6·(0.44) = 0.44
……………
is_first_class0.5{0,1,0}{1,1,0}3/6·(0.44)+3/6·(0.44)=0.44

The winner on this toy data is age ≤ 24 with score ≈ 0.25. bestSplit returns {feature_index=0, threshold=24, score=0.25, valid=true}.

Step 3: buildTree partitions and recurses

  • left slice: {(22,0), (8,0)}, labels {0, 0}
  • right slice: {(38,1), (26,0), (35,1), (54,1)}, labels {1,1,1,0}

Root becomes an internal node: feature_index=0, threshold=24. Recurse into children at depth 1.

Left child: buildTree({(22,0),(8,0)}, {0,0}, 1, 3, 2)

  • gini({0,0}) = 0 → pure → make a leaf with prediction = leafPrediction({0,0}) = 0.

Right child: buildTree({(38,1),(26,0),(35,1),(54,1)}, {1,1,1,0}, 1, 3, 2)

  • Not at max_depth. 4 samples ≥ min_samples. gini = 1-(0.25²+0.75²)=0.375.
  • Calls bestSplit again → finds some splitting question on age or class → builds deeper nodes.

Step 4: prediction

Call tree.predict({30, 1}).

  1. traverse(root, {30, 1}). Root is internal. feature_index=0, threshold=24. sample[0] = 30. 30 ≤ 24? No. Go right.
  2. Recurse into the right subtree's root. Whatever question it asks (maybe "is_first_class ≤ 0.5?"), evaluate and pick left/right.
  3. Keep going until you hit a leaf. Return leaf->prediction.

That's the entire lifecycle. Construct, fit, predict.

7. Glossary

Class / class label
The category a sample belongs to. For Titanic: 0 (died) or 1 (survived).
Feature
One column of X. A numeric attribute of a sample.
Gini impurity
1 - Σ pᵢ² where pᵢ is the fraction of samples in class i. 0 means pure, higher means more mixed.
Greedy algorithm
An algorithm that makes the locally best choice at each step without looking ahead. Tree building is greedy: at each node we pick the split with the best immediate Gini improvement.
Impurity
Any measure of how mixed-up a group's labels are. Gini is one such measure; there are others (entropy, classification error) but this file uses Gini.
Label / target / y
The value we want to predict. One per sample.
Leaf
A tree node that has no children and outputs a fixed prediction.
Node
One box in the tree. Either internal (has a question and two children) or a leaf.
Overfitting
When a model learns the training data so specifically that it fails on new data. Very deep trees overfit; max_depth and min_samples fight this.
Prediction / inference
Using a trained model to produce a label for a new sample.
Recursion
A function that calls itself. buildTree recurses into its own left and right subtrees. traverse recurses down the tree during prediction.
Root
The top node of the tree. Every prediction starts there.
Sample / row / observation
One example. One row of X with its corresponding label in y.
Split
Partitioning a group of samples into two based on a (feature, threshold) pair.
Threshold
The numeric cutoff in a split's question.
Training / fitting
The process of learning the tree from labeled data. Done by fit().
Weighted Gini
The combined Gini of two split groups, weighted by their sizes. The score we minimize when searching for the best split.
