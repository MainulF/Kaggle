1. What We Are Building

A Decision Tree classifies samples by asking a series of yes/no questions about feature values. Each question has the form: "Is feature[j] ≤ threshold?" Samples that answer yes travel left; samples that answer no travel right. This continues until a sample reaches a leaf node, which outputs the final prediction.

Key Insight
A decision tree is a binary tree. Internal nodes store split rules. Leaf nodes store predictions. Building the tree = choosing the best splits recursively. Predicting = walking from the root to a leaf.

Here is the overall build order you will follow in this guide:

  1. Define a Node struct to hold split rules and child pointers.
  2. Implement gini() to measure how mixed a set of labels is.
  3. Implement weightedGini() to score candidate splits.
  4. Implement bestSplit() to search all features and thresholds.
  5. Implement buildTree() to recursively grow the tree.
  6. Implement fit() and predict() as the public interface.

2. The Node Structure

A Node is the single unit the tree is made of. Every node is either an internal node (stores a split rule) or a leaf node (stores a final prediction). One struct handles both roles.

What an internal node needs

  • feature_index — which column of the data matrix to split on (e.g., column 2).
  • threshold — the value to compare against (e.g., age ≤ 30.5).
  • left — pointer to the child that receives samples where feature ≤ threshold.
  • right — pointer to the child that receives samples where feature > threshold.

What a leaf node needs

  • prediction — the class label to output for any sample that reaches this leaf.
  • is_leaf — a boolean flag so traversal code knows to stop here.
Memory Management
Use std::unique_ptr<Node> for children, not raw pointers. unique_ptr owns the child and destroys it automatically when the parent is destroyed. This means you never need to write a destructor for the tree — the entire tree cleans itself up when root_ goes out of scope.

Definition

struct Node {
    int feature_index = -1;   // which feature column to test
    double threshold = 0.0;  // value to compare against
    double prediction = 0.0;  // used only by leaf nodes
    bool is_leaf = false;
    std::unique_ptr<Node> left;
    std::unique_ptr<Node> right;
};
Best Practice
Place this struct inside the DecisionTree class as a private nested struct. It is an implementation detail — callers should never see or touch a raw Node. Giving all members default values means a freshly constructed node is always in a valid state.

3. Gini Impurity

Gini impurity measures how mixed a set of labels is. A value of 0.0 means all labels are the same class (a pure node). Higher values mean more mixing. For binary classification the maximum is 0.5.

Formula

Let pi be the proportion of class i in the label set:

G = 1 − Σ(pi²)

For example, if a node has 6 samples of class 0 and 4 of class 1: p0 = 0.6, p1 = 0.4 → G = 1 − (0.36 + 0.16) = 0.48. A perfectly pure node: p0 = 1.0 → G = 1 − 1.0 = 0.0.

Function Signature

static double gini(const std::vector<double>& labels);

It is static because it only depends on its argument — no tree state is needed.

Steps to Implement

  1. If labels is empty, return 0.0.
  2. Count the number of occurrences of each unique label. A std::map<double, int> is a natural fit: iterate labels and increment counts[label] for each element.
  3. Compute n = labels.size() (store as double to avoid integer division later).
  4. For each entry in your counts map, compute p = count / n, then accumulate sum += p * p.
  5. Return 1.0 - sum.
Fix Your Current Code
Your current early-return condition — all_of(x > 0) — is logically wrong. It returns 0.0 whenever all values are positive, which is almost always true for class labels. The correct early exit is when the node is pure: all labels are equal. You can check this with std::ranges::all_of(labels, [&](double x){ return x == labels[0]; }).

Also: you do not need to sort the data for Gini impurity. Sorting is used for the Gini coefficient (an economics measure) — a different concept. Remove the sort.

4. Weighted Gini

When you test a candidate split, you end up with two child groups: left and right. The quality of the split is the weighted average of their Gini scores, weighted by the number of samples in each group.

A 999/1 split should be judged very differently from a 500/500 split, even if the raw Gini scores are the same. Weighting by size captures this.

Formula

weighted_gini = (|left| / |total|) × gini(left) + (|right| / |total|) × gini(right)

Function Signature

static double weightedGini(
    const std::vector<double>& left_labels,
    const std::vector<double>& right_labels
);

Steps to Implement

  1. Compute n_total = (double)(left_labels.size() + right_labels.size()). Guard: if n_total == 0, return 0.0.
  2. Compute w_left = left_labels.size() / n_total.
  3. Compute w_right = right_labels.size() / n_total.
  4. Return w_left * gini(left_labels) + w_right * gini(right_labels).
Note
Lower weighted Gini = better split. A split that produces two pure children has weighted Gini = 0.0 — the best possible score.

5. The SplitResult Helper

bestSplit() searches through many candidate splits and needs to return the winner. Bundle the result in a small struct so the return type is self-documenting and easy to work with.

struct SplitResult {
    int feature_index = -1;
    double threshold = 0.0;
    double score = 1.0;  // weighted Gini; lower is better
    bool valid = false;// false = no useful split found
};

score is initialised to 1.0 (the theoretical maximum) so that any real improvement will be detected by a simple < comparison. valid = false is your sentinel for "we could not find any split for this node."

Like Node, place this struct inside the DecisionTree class as a private nested type.

6. Splitting the Data

Before evaluating any split, you need a helper that partitions rows of X (and their corresponding labels y) into two groups based on a feature and a threshold.

Rule

  • Row i goes left if X[i][feature_index] ≤ threshold.
  • Row i goes right if X[i][feature_index] > threshold.

Function Signatures

// Returns the subset of (X, y) where feature ≤ threshold
static std::pair<std::vector<std::vector<double>>, std::vector<double>>
splitLeft(
    const std::vector<std::vector<double>>& X,
    const std::vector<double>& y,
    int feature_index,
    double threshold
);

// Same, but for feature > threshold
static std::pair<std::vector<std::vector<double>>, std::vector<double>>
splitRight(
    const std::vector<std::vector<double>>& X,
    const std::vector<double>& y,
    int feature_index,
    double threshold
);

Steps to Implement (shown for splitLeft; splitRight is a one-character change)

  1. Create empty vectors X_out and y_out.
  2. Loop over every index i from 0 to X.size().
  3. If X[i][feature_index] ≤ threshold, append X[i] to X_out and y[i] to y_out.
  4. Return std::make_pair(X_out, y_out).
Design Tip
You can merge both functions into one that fills left and right in the same loop, avoiding iterating the data twice per candidate threshold. Either approach is correct — choose whichever is clearest to you.

7. Finding the Best Split

This is the core of the learning algorithm. For every feature and every candidate threshold, evaluate the resulting split with weightedGini() and track the best one seen so far.

How candidate thresholds are chosen

For a given feature j, collect all values that appear in column j of X. Sort them and remove duplicates. For each pair of consecutive unique values (a, b), use their midpoint (a + b) / 2.0 as a threshold candidate. This guarantees you test every meaningful binary split for that feature.

Function Signature

static SplitResult bestSplit(
    const std::vector<std::vector<double>>& X,
    const std::vector<double>& y
);

Steps to Implement

  1. Create a default SplitResult best (score = 1.0, valid = false).
  2. Determine the number of features: n_features = X[0].size().
  3. For each feature index j in range [0, n_features):
    1. Extract every value in column j into a temporary vector.
    2. Sort it, then use std::unique to remove duplicates.
    3. For each consecutive pair of unique values (vals[k], vals[k+1]), compute threshold = (vals[k] + vals[k+1]) / 2.0.
    4. Call splitLeft and splitRight to get the two label subsets.
    5. Skip this threshold if either subset is empty (no meaningful split).
    6. Compute score = weightedGini(left_labels, right_labels).
    7. If score < best.score, update best: set feature_index = j, threshold, score, and valid = true.
  4. Return best.
Time Complexity
This is O(n² × f) — for every feature, you try up to n thresholds, each requiring an O(n) partition pass. For the Titanic dataset this is fast enough. For larger datasets, Random Forests speed this up by randomly subsampling features at each split.

8. Leaf Prediction

When a node cannot or should not be split further, it becomes a leaf. Its job is to output a single prediction for every sample that reaches it. For classification, this is a majority vote: the most common class label in the node's training samples.

Function Signature

static double leafPrediction(const std::vector<double>& labels);

Steps to Implement

  1. Use a std::map<double, int> to count occurrences of each label — same approach as in gini().
  2. Use std::max_element on the map. You will need a custom comparator that compares the .second (count) of each map entry.
  3. Return the .first (label) of the winning entry.

9. Building the Tree Recursively

This function is the engine of the whole algorithm. It receives a subset of the data, decides whether to create a leaf or an internal node, and — in the recursive case — calls itself twice to grow the subtrees.

Function Signature

std::unique_ptr<Node> buildTree(
    std::vector<std::vector<double>> X,  // by value — each call owns its slice
    std::vector<double> y,
    int depth
);
Why pass X and y by value?
Each recursive call works on its own slice of data. Passing by value makes ownership explicit and avoids subtle bugs where two calls share the same vector. The copies are going to happen anyway because each subtree sees a different subset of rows.

Step A — Base Cases (when to make a leaf)

Check these conditions before calling bestSplit. If any is true, create and return a leaf node.

  1. depth ≥ max_depth_ — the tree has reached its maximum allowed depth.
  2. y.size() < (size_t)min_samples_ — too few samples to justify a split.
  3. gini(y) == 0.0 — the node is already pure; splitting would gain nothing.
  4. bestSplit(X, y).valid == false — no split improves purity (all features have only one unique value in this subset).

To make a leaf: create a std::make_unique<Node>(), set is_leaf = true, set prediction = leafPrediction(y), and return it.

Order Matters
Always check cases 1–3 before calling bestSplit(), because bestSplit accesses X[0].size(). If X is empty that is undefined behavior. Checking y.size() first protects you.

Step B — Recursive Case

  1. Call SplitResult best = bestSplit(X, y). If !best.valid, fall through to making a leaf (same as base case 4 above).
  2. Call splitLeft and splitRight using best.feature_index and best.threshold to partition X and y.
  3. Create a new internal node via std::make_unique<Node>(). Set its feature_index and threshold from best. Leave is_leaf = false.
  4. Set node->left = buildTree(X_left, y_left, depth + 1).
  5. Set node->right = buildTree(X_right, y_right, depth + 1).
  6. Return the node.

10. The Public Interface

Constructor

DecisionTree(int max_depth = 5, int min_samples = 2);

Store the arguments into max_depth_ and min_samples_. root_ will be nullptr by default (since unique_ptr default-constructs to null).

fit()

void fit(
    const std::vector<std::vector<double>>& X,
    const std::vector<double>& y
);

One line: root_ = buildTree(X, y, 0). The recursion in buildTree handles everything from here.

traverse() — private helper

double traverse(const Node* node, const std::vector<double>& sample) const;

Steps to Implement traverse()

  1. If node->is_leaf is true, return node->prediction.
  2. Check sample[node->feature_index] ≤ node->threshold. If true, recurse: return traverse(node->left.get(), sample).
  3. Otherwise, recurse right: return traverse(node->right.get(), sample).

predict() — public

double predict(const std::vector<double>& sample) const;

Assert that root_ is not null (i.e., fit() was called first), then return traverse(root_.get(), sample).

11. Full Class Skeleton

Here is the complete skeleton with all types and signatures in place but no function bodies. This is what your header should look like when all the pieces are wired together. Fill each body one function at a time, in the order listed in the recommended build order. 

#pragma once
#include <vector>
#include <map>
#include <memory>
#include <algorithm>
#include <numeric>
#include <cassert>

class DecisionTree {
public:
    DecisionTree(int max_depth = 5, int min_samples = 2);

    void   fit(const std::vector<std::vector<double>>& X,
               const std::vector<double>& y);

    double predict(const std::vector<double>& sample) const;

private:
    // ── nested types ──────────────────────────────────────────────────
    struct Node {
        int    feature_index = -1;
        double threshold     = 0.0;
        double prediction    = 0.0;
        bool   is_leaf       = false;
        std::unique_ptr<Node> left;
        std::unique_ptr<Node> right;
    };

    struct SplitResult {
        int    feature_index = -1;
        double threshold     = 0.0;
        double score         = 1.0;   // weighted Gini; lower is better
        bool   valid         = false;
    };

    // ── hyperparameters ───────────────────────────────────────────────
    int max_depth_;
    int min_samples_;

    // ── tree state ────────────────────────────────────────────────────
    std::unique_ptr<Node> root_;

    // ── private helpers (implement in this order) ─────────────────────
    static double gini(const std::vector<double>& labels);

    static double weightedGini(const std::vector<double>& left_labels,
                               const std::vector<double>& right_labels);

    static std::pair<std::vector<std::vector<double>>, std::vector<double>>
    splitLeft(const std::vector<std::vector<double>>& X,
              const std::vector<double>& y,
              int feature_index, double threshold);

    static std::pair<std::vector<std::vector<double>>, std::vector<double>>
    splitRight(const std::vector<std::vector<double>>& X,
               const std::vector<double>& y,
               int feature_index, double threshold);

    static double leafPrediction(const std::vector<double>& labels);

    static SplitResult bestSplit(const std::vector<std::vector<double>>& X,
                                 const std::vector<double>& y);

    std::unique_ptr<Node> buildTree(std::vector<std::vector<double>> X,
                                    std::vector<double> y,
                                    int depth);

    double traverse(const Node* node,
                    const std::vector<double>& sample) const;
};
Recommended Build Order
  1. Implement and manually test gini() with a few hand-crafted vectors.
  2. Implement weightedGini().
  3. Implement splitLeft() and splitRight().
  4. Implement leafPrediction().
  5. Implement bestSplit() — it composes all of the above.
  6. Implement buildTree() — it composes everything.
  7. Wire up fit(), traverse(), and predict().
Build bottom-up: get each helper working in isolation before composing them. The tree assembles naturally once the leaf-level functions are solid.
